# Flowline

SaaS-приложение для управления проектами с диаграммой Ганта: задачи и подзадачи,
зависимости, статусы и приоритеты, рабочий календарь (5-дневная неделя),
история изменений, отделы и команда.

## Стек

- **Backend:** Node.js (Express), PostgreSQL, JWT в httpOnly cookie
- **Frontend:** React + Vite, Tailwind CSS
- **Деплой:** Docker Compose (Postgres + backend + frontend за nginx)

## Быстрый старт

Требуется установленный [Docker](https://docs.docker.com/get-started/get-docker/) и
[Docker Compose](https://docs.docker.com/compose/install/) (идёт в комплекте с Docker Desktop).

```bash
git clone https://github.com/aspacev1/flowline.git
cd flowline

# опционально: настроить секреты под продакшен (JWT, OAuth, SMTP — см. раздел ниже)
cp .env.example .env

docker compose up --build
```

После того как все контейнеры поднялись (обычно 30–60 секунд на первый запуск,
пока строятся образы и применяется схема БД), откройте:

**http://localhost:8080**

Можно также зарегистрировать новый аккаунт по email/паролю или через Google/Microsoft/Apple

(имя, должность, название компании) и становится владельцем новой организации; дальше он
может пригласить коллег по ссылке или по email — см. [«Приглашение в организацию»](#приглашение-в-организацию).

## Приглашение в организацию

Любой участник организации может пригласить коллег двумя способами:

- **Ссылка-приглашение** — одна постоянная ссылка на организацию, которую можно
  скопировать и переслать. При необходимости её можно сгенерировать заново
  (старая ссылка перестанет работать).
- **По email** — ввести один или несколько email-адресов; каждому будет отправлено
  письмо со ссылкой-приглашением (требует настроенный SMTP, см. ниже).

Перешедший по ссылке пользователь регистрируется (email/пароль или OAuth) и
автоматически присоединяется к организации пригласившего — шаг с названием
компании при этом пропускается.

## Остановка и сброс данных

```bash
# остановить контейнеры (данные останутся)
docker compose down

# остановить и стереть все данные БД (следующий запуск пересоздаст схему и демо-данные)
docker compose down -v
```

## Переменные окружения

См. `.env.example`. Главное, что нужно поменять перед реальным продакшен-деплоем:

- `JWT_SECRET` — длинная случайная строка (например, `openssl rand -hex 32`)
- `FRONTEND_PORT` — порт, на котором будет доступен сайт снаружи
- `FRONTEND_ORIGIN` — публичный адрес фронтенда (для CORS)

Опционально (по умолчанию выключено, приложение работает и без этого):

- **OAuth-вход** — `OAUTH_REDIRECT_BASE_URL` плюс пара `CLIENT_ID`/`CLIENT_SECRET` для
  `GOOGLE_*`, `MICROSOFT_*`, либо `APPLE_CLIENT_ID`/`APPLE_TEAM_ID`/`APPLE_KEY_ID`/
  `APPLE_PRIVATE_KEY` для Apple. Кнопка входа показывается только для тех провайдеров,
  у которых заполнены все нужные переменные. `APPLE_PRIVATE_KEY` — многострочный ключ;
  записывайте его в одну строку с `\n` вместо переводов строк (пример — в `.env.example`).
- **Отправка email-приглашений** — `SMTP_HOST`, `SMTP_PORT`, `SMTP_USER`, `SMTP_PASS`,
  `SMTP_FROM`. Без них приглашение по ссылке всё равно работает, но приглашение по
  email вернёт ошибку «отправка почты не настроена».

## Известные ограничения текущей версии

- Рабочая неделя зафиксирована как 5 дней (пн–пт), без поддержки праздников
  или индивидуальных графиков — это сознательное упрощение, см. историю обсуждения.
- Один ответственный на задачу (не many-to-many) — менеджеров и коллабораторов
  на проекте может быть неограниченное количество, но конкретная задача закреплена
  за одним человеком.
- Роли участников организации/проекта хранятся в базе, но прав доступа по ролям
  пока нет — любой участник организации может управлять любым проектом и приглашениями.
