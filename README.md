# Flowline

SaaS-приложение для управления проектами с диаграммой Ганта: задачи и подзадачи,
зависимости, статусы и приоритеты, рабочий календарь (5-дневная неделя),
история изменений, отделы и команда.

## Стек

- **Backend:** Node.js (Express), PostgreSQL, JWT в httpOnly cookie
- **Frontend:** React + Vite, Tailwind CSS
- **Деплой:** Docker Compose (Postgres + backend + frontend за nginx)

## Быстрый старт

Требуется установленный [Docker](https://docs.docker.com/get-docker/) и Docker Compose
(идёт в комплекте с Docker Desktop).

```bash
git clone <repo-url> flowline
cd flowline

# опционально: настроить секреты под продакшен
cp .env.example .env

docker compose up --build
```

После того как все контейнеры поднялись (обычно 30–60 секунд на первый запуск,
пока строятся образы и применяется схема БД), откройте:

**http://localhost:8080**

## Структура проекта

```
flowline/
├── backend/          # Express API
│   ├── src/
│   │   ├── routes/        # auth, projects, work-items, team
│   │   ├── middleware/    # JWT-аутентификация
│   │   ├── utils/         # расчёт рабочих дней
│   │   ├── db/             # пул соединений Postgres
│   │   ├── server.js
│   │   └── migrate.js      # применяет schema.sql + seed.sql при первом старте
│   └── Dockerfile
├── frontend/          # React SPA
│   ├── src/
│   │   ├── components/     # AuthScreen, ProjectsScreen, GanttScreen, TeamScreen
│   │   ├── api.js           # клиент для backend API
│   │   ├── dateUtils.js     # та же логика рабочих дней, что на backend
│   │   └── constants.js     # справочники статусов/приоритетов
│   ├── nginx.conf            # проксирует /api на backend, раздаёт статику
│   └── Dockerfile
├── db/
│   ├── schema.sql
│   └── seed.sql
└── docker-compose.yml
```

## Остановка и сброс данных

```bash
# остановить контейнеры (данные останутся)
docker compose down

# остановить и стереть все данные БД (следующий запуск пересоздаст схему и демо-данные)
docker compose down -v
```

## Переменные окружения

См. `.env.example`. Главное, что нужно поменять перед реальным продакшен-деплоем:

- `JWT_SECRET` — длинная случайная строка (например, `openssl rand -hex 32`)
- `FRONTEND_PORT` — порт, на котором будет доступен сайт снаружи
- `FRONTEND_ORIGIN` — публичный адрес фронтенда (для CORS)

## Известные ограничения текущей версии

- Рабочая неделя зафиксирована как 5 дней (пн–пт), без поддержки праздников
  или индивидуальных графиков — это сознательное упрощение, см. историю обсуждения.
- Один ответственный на задачу (не many-to-many) — менеджеров и коллабораторов
  на проекте может быть неограниченное количество, но конкретная задача закреплена
  за одним человеком.
- Регистрация создаёт новую организацию на каждого пользователя; функциональности
  "пригласить в существующую организацию" пока нет.
